# MiniApp 开仓页面 (Open Position) 移动端 UI/UX 优化总结

## 优化目标
解决由过多的说明文字、巨大的 Padding、突兀的边框嵌套导致的开仓页面“臃肿”、“竖直空间浪费过大”、“设计不统一”的问题。提升移动端紧凑度与现代化高级感。

## 优化进展 - 已完成核心模块重排
本日重构主要聚焦在 `e:\goProject\TgLpBot\miniapp\src\App.jsx` 中的深层结构：

### 1. 表单与核心输入模块（Amount & Slippage）
- **合并多余卡片**：使用原生的无框结构将“开仓金额”与“滑点”从巨大外边框的卡片中释放。
- **iOS 风格内联输入框**：输入框采用 `pl-3 pr-11` 处理，将单位 (`USDT`, `%`) 绝对定位于输入框内右侧，极大减小了原先独立分为多行的冗余感。
- **推荐策略微型化**：智能提示卡片（稳、均、进）转化为微型气泡标签紧贴金额输入。

### 2. 区间设置模块全面去冗（Range Settings）
- **胶囊切换器**：百分比区间及价格模式之间的老式笨重切换大按钮，彻底改为精巧的 Segmented Control (分段选择器)。
- **微型化快捷区间**：如 `1%`、`3%` 的大范围按钮通过 `inline-flex` 改为更小巧的单行水平可滑动的按钮池组合。
- **清除多余文字教学**：通过合理的 Placeholder 提示（“下限”、“上限”），除去了原本大段的繁冗教学文本，界面回归沉浸与安静感。

### 3. 操作选项扁平化（Task Modes & DCA）
- **本次开仓 (Task Modes)**：剥离多余的渐变边框卡片。利用紧凑横向滑动、激活态黑色背景、默认态半透明的高级 UI 对比。
- **分批开仓 (DCA)**：引入了全新的“一键展开/折叠”组件。核心信息（笔数聚合数据）被吸顶在一行内；下方配置部分将输入框缩小，把独立提示统合成内联标签以适应单手操作的紧缩。

## 后续建议
后续若需进一步体验提升，可通过增加 `Framer Motion` 的丝滑展开收起高度动画，令 `BottomSheet` 在折叠/展开“分批开仓”时拥有更平顺的缓冲效果。本期更新的代码已落实并能稳定呈现，达到了用户要求的“设计统一，紧凑，专业”的目标。
