# 监控通知：费率/活跃费率统一 & MiniApp 同步改造

**日期**：2026-03-26

## 问题背景

池子监控的费率和活跃费率筛选与热门池子展示的数据不一致：
- **费率**：后端用 `pool.PoolMFeeRate`（Uniswap bps 整数）对比，但热门池子用 `fees/tvl*100` 动态计算
- **活跃费率**：后端用 `pool.ActiveLiquidityRatio`（小数比例）对比，但热门池子用 `fees/activeLiquidityUSD*100` 动态计算
- 前后端单位不匹配：前端让用户输百分比，但后端存储/对比的格式各不相同

## 改动内容

### 后端
| 文件 | 改动 |
|------|------|
| `models/smart_money_golden_dog.go` | `PoolMinFeeRate` 从 `int` 改为 `float64`（存储百分比） |
| `service/smart_money_golden_dog/service.go` | 费率筛选改为 `fees/tvl*100`，活跃费率改为 `fees/activeLiquidityUSD*100`，与热门池子完全一致 |
| `web_server/smart_money_golden_dog_config.go` | API 类型 `*int` → `*float64`，clamp 函数和类型断言同步修改 |
| `service/smart_money_golden_dog/service_test.go` | 测试用例改用 `ActiveLiquidityUSD` 字段和百分比阈值 |

### 前端 WebApp
| 文件 | 改动 |
|------|------|
| `SmartMoneyDashboard.jsx` | 移除活跃费率的 `×100/÷100` 转换（后端已改为百分比存储） |

### 前端 MiniApp
| 文件 | 改动 |
|------|------|
| `SmartMoneyPage.jsx` | 1) 费率从 select 下拉框改为 number input; 2) 移除 `min_total_fees` 字段; 3) 精简为 5 个字段（TVL/VOL/笔数/费率/活跃费率）; 4) StatCards 更新; 5) 说明文字更新; 6) 移除 `×100/÷100` 转换 |

## 数据流（改后）

```
用户输入 0.005（表示 0.005%）
  → 前端直接传 0.005 到后端
  → 后端存储 float64: 0.005
  → 筛选时：poolFeeRate = pool.TotalFees / pool.CurrentPoolValue * 100
  → 对比：poolFeeRate >= 0.005 → 匹配
```
