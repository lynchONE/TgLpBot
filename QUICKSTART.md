# Quick Start Guide

Get your TgLpBot up and running in minutes!

## Prerequisites

- Go 1.21+
- MySQL 8.0+
- Redis 6.0+
- Telegram Bot Token
- BSC RPC endpoint (or use public ones)

## 5-Minute Setup

### 1. Clone and Setup

```bash
# Clone the repository
git clone <your-repo-url>
cd TgLpBot

# Run setup script
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### 2. Configure Environment

Edit `.env` file with your settings:

```bash
# Required: Get from @BotFather on Telegram
TELEGRAM_BOT_TOKEN=your_bot_token_here

# Database (use defaults for local development)
MYSQL_PASSWORD=your_secure_password

# Encryption key (already generated by setup script)
# ENCRYPTION_KEY=... (don't change)

# Optional: OKX DEX API (for optimal swap routes)
# OKX_API_KEY=...
# OKX_SECRET_KEY=...
# OKX_PASSPHRASE=...
```

### 3. Deploy Zap Contract

```bash
cd contracts
npm install

# For testnet (recommended for testing)
npm run deploy:testnet

# For mainnet (production)
npm run deploy:mainnet
```

Copy the deployed contract address and add to `.env`:

```bash
ZAP_CONTRACT_ADDRESS=0x...
```

### 4. Run the Bot

```bash
# Back to project root
cd ..

# Run the bot
make run
```

That's it! Your bot is now running! 🎉

## Using Docker (Alternative)

If you prefer Docker:

```bash
# Create .env file
cp .env.example .env
# Edit .env with your settings

# Start all services
docker-compose up -d

# View logs
docker-compose logs -f bot
```

## Common Issues

### Bot not responding

```bash
# Check if bot is running
ps aux | grep tglpbot

# Check logs
tail -f /var/log/tglpbot/output.log
```

### Database connection error

```bash
# Check MySQL is running
sudo systemctl status mysql

# Test connection
mysql -u root -p -e "SHOW DATABASES;"
```

### Redis connection error

```bash
# Check Redis is running
redis-cli ping
# Should return: PONG
```

### Transaction fails

- Ensure wallet has enough BNB for gas
- Check token approvals
- Verify Zap contract address is correct
- Try increasing slippage tolerance

## Production Deployment

For production deployment:

```bash
# Build and deploy
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

This will:
- Build the application
- Install as systemd service
- Start the bot automatically
- Enable auto-restart on failure

## Security Checklist

Before going to production:

- [ ] Use strong MySQL password
- [ ] Secure your encryption key
- [ ] Never commit `.env` to git
- [ ] Use firewall to restrict database access
- [ ] Get Zap contract audited
- [ ] Test with small amounts first
- [ ] Enable 2FA on your server
- [ ] Regular backups of database
- [ ] Monitor logs for suspicious activity
