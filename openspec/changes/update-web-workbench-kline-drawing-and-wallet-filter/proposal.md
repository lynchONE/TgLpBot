# Change: 增强 Web Workbench K 线测量工具与钱包气泡筛选

## Why
- 当前 `webapp` 的自绘 K 线已经能展示聪明钱气泡和区间虚线，但用户仍然缺少两类高频操作能力：
  - 在图表上快速量化某一段价格区间的涨跌幅，而不需要手动估算。
  - 在气泡较多时，只看满足金额条件或指定钱包的气泡，避免图面过于拥挤。
- 这两项能力都属于 K 线工作台内部交互增强，不需要新增后端接口，适合在现有 `webapp` 图表模块上增量实现。

## What Changes
- 为 Web Workbench K 线新增两种测量工具：
  - 直线测量：用户依次选择起点和终点后，显示两点之间的价格涨跌幅百分比。
  - 矩形测量：用户拖拽出一个框选区域后，显示该价格区间的涨跌幅百分比。
- 在 K 线左上角新增“钱包气泡筛选”下拉面板，支持：
  - 按开单金额阈值过滤气泡。
  - 按钱包多选过滤气泡。
  - 全选、清空、重置等快捷操作。
- 筛选结果仅影响 K 线上的聪明钱气泡与相关详情入口，不改变 K 线蜡烛数据本身。

## Scope Assumption
- V1 的测量工具以“读数”为目标，不做趋势线、斐波那契、持久化保存、多图层管理等 TradingView 级复杂绘图。
- V1 同一时间只保留一个当前测量结果；用户切换工具、重新绘制或点击清除后，旧测量结果被替换。
- 钱包筛选范围基于当前已加载的 K 线 marker 数据，不额外请求后端拉取完整钱包全集。
- “开单金额”使用现有 marker 数据中的 `estimated_usd` 作为筛选依据。

## Impact
- Affected specs:
  - `web-workbench`
- Affected code:
  - `webapp/src/App.jsx`
  - `webapp/src/components/KlineChart.jsx`
  - `webapp/src/styles.css`
- Risks / tradeoffs:
  - 图表上叠加绘图层后，需谨慎处理与现有 hover tooltip、marker 点击、拖拽缩放之间的事件冲突。
  - 钱包筛选项来自当前 marker 数据，因此切池子、切周期或可视区间变化时，筛选候选钱包会动态变化。
  - 若金额阈值与钱包勾选组合过严，图面可能没有任何气泡，需要明确展示“已过滤为空”的状态。
