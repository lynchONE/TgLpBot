## ADDED Requirements

### Requirement: 开仓界面打开时 MUST 启动池子预检测
系统 MUST 支持在用户打开开仓界面后，基于链、池子和当前钱包上下文立即执行与金额、区间和滑点无关的基础检测，而不是等待表单填写完整后才开始。

#### Scenario: 用户刚打开开仓界面时立即开始预检测
- **GIVEN** 用户已经选中池子并打开开仓界面
- **WHEN** 客户端初始化开仓流程
- **THEN** 系统 MUST 立即发起池子预检测
- **AND** MUST 返回基础池子信息、基础风控结果和可供后续复用的上下文
- **AND** MUST NOT 依赖用户先输入金额、区间或滑点

#### Scenario: 多钱包模式下先做池子侧检查
- **GIVEN** 当前用户开启了多钱包模式且尚未完成钱包选择
- **WHEN** 用户先打开开仓界面
- **THEN** 系统 MUST 先完成与池子本身相关的基础检测
- **AND** MUST 在钱包确定后补齐钱包相关检查
- **AND** MUST 向客户端明确哪些检查仍待钱包上下文完成

### Requirement: 开仓预览 MUST 增量复用预检测结果
系统 MUST 让金额、区间和滑点输入后的预览优先复用预检测阶段已经得到的池子元数据与基础检查结果，只重新计算与这些输入直接相关的部分。

#### Scenario: 用户输入金额后仅补算金额相关检查
- **GIVEN** 当前池子已经完成预检测
- **WHEN** 用户首次输入或修改开仓金额
- **THEN** 系统 MUST 复用已有池子元数据和基础检查结果
- **AND** MUST 只重新计算与金额和区间相关的风险结果、entry swap 预览和报价

#### Scenario: 用户修改滑点后刷新报价而不是重做整套池子检测
- **GIVEN** 当前池子和钱包的基础检测已经完成
- **WHEN** 用户修改任务滑点或 entry swap 滑点
- **THEN** 系统 MUST 优先刷新与滑点相关的报价和预估结果
- **AND** MUST NOT 无条件重新执行整套池子基础检测

### Requirement: 预检测结果 MUST 可供最终执行复用
系统 MUST 允许最终 `open_position` 执行阶段复用同一轮预检测的上下文，以减少重复 RPC，同时在上下文失效时自动回退到完整校验。

#### Scenario: prepare 上下文有效时执行直接复用
- **GIVEN** 用户已经完成开仓预检测和后续预览
- **AND** 相关上下文仍在有效期内
- **WHEN** 用户提交最终开仓
- **THEN** 系统 MUST 复用该上下文中的池子元数据和基础检查结果
- **AND** MUST 只补充执行前必须重新确认的动态条件

#### Scenario: prepare 上下文失效时自动回退
- **GIVEN** 用户提交最终开仓时 prepare 上下文已经失效或与当前参数不匹配
- **WHEN** 系统处理 `open_position`
- **THEN** 系统 MUST 自动回退到完整校验和完整数据加载
- **AND** MUST 保持旧客户端与异常场景下的执行兼容性
