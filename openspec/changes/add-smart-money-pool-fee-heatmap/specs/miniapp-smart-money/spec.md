## ADDED Requirements
### Requirement: 聪明钱池子视图必须提供活跃池子与收益火焰图子页
聪明钱模块的池子视图 MUST 提供 `活跃池子` 与 `收益火焰图` 两个子页。现有池子列表的内容、筛选、分页、池子详情入口和快捷跟单入口 MUST 保留在 `活跃池子` 子页中。

#### Scenario: 用户打开池子视图
- **WHEN** 用户进入聪明钱模块的池子视图
- **THEN** 页面展示 `活跃池子` 与 `收益火焰图` 两个子页入口
- **AND** `活跃池子` 子页展示当前已有的池子列表能力

#### Scenario: 用户切换到收益火焰图
- **WHEN** 用户点击 `收益火焰图`
- **THEN** 页面展示按池子聚合的手续费收益热力视图
- **AND** 不移除或改变 `活跃池子` 中的原有池子列表内容

### Requirement: 收益火焰图必须支持手续费与速率两种口径
收益火焰图 MUST 支持 `手续费` 与 `速率` 两种展示/排序口径。`手续费` 口径 MUST 按池子下聪明钱活跃仓位当前未领取手续费 USD 总和排序。`速率` 口径 MUST 基于活跃仓位从开仓时间到当前时间的平均手续费速度，并同时考虑手续费、仓位金额和时间窗口，用于比较单位资金在单位时间内吃手续费的速度。

#### Scenario: 按手续费排序
- **WHEN** 用户选择 `手续费` 口径
- **THEN** 系统按池子下聪明钱活跃仓位的手续费 USD 总和从高到低展示
- **AND** 每个池子 MUST 展示手续费总额

#### Scenario: 按速率排序
- **WHEN** 用户选择 `速率` 口径
- **THEN** 系统按归一化手续费速率从高到低展示
- **AND** 每个池子 MUST 展示选定窗口折算手续费、参与计算的仓位金额和归一化速率

### Requirement: 收益火焰图必须支持固定时间窗口
收益火焰图 MUST 支持 `30s`、`1m`、`5m`、`1h` 四个固定时间窗口。速率计算 MUST 使用用户当前选择的窗口。

#### Scenario: 用户切换时间窗口
- **WHEN** 用户在收益火焰图中从 `1m` 切换到 `5m`
- **THEN** 系统使用 `5m` 窗口重新加载或重新计算池子收益排行
- **AND** 当前选择的 `手续费` 或 `速率` 口径保持不变

#### Scenario: 数据质量不足
- **WHEN** 某池子的活跃仓位缺少有效手续费、仓位金额或开仓时间
- **THEN** 系统 MUST 返回该池子参与速率计算的仓位数量和缺失数据计数
- **AND** 前端 MUST 可见地弱化或提示该状态，不得把缺失数据展示为可靠收益

### Requirement: 收益火焰图必须支持快捷跟单
收益火焰图中的每个池子 MUST 提供快捷跟单入口，并复用当前聪明钱池子开仓流程和既有开仓风控。

#### Scenario: 用户从火焰图快捷跟单
- **WHEN** 用户点击某个火焰图池子的 `跟单` 操作
- **THEN** 系统打开现有开仓入口并预填该池子的链、协议、池子地址、交易对和费率信息
- **AND** 后续开仓必须继续执行现有风控与预览流程

### Requirement: 双端聪明钱池子火焰图口径必须一致
MiniApp 与 WebApp 的聪明钱池子视图 MUST 使用一致的后端接口字段、窗口定义、手续费口径和速率口径。

#### Scenario: 同一窗口同一排序口径
- **WHEN** MiniApp 与 WebApp 都选择 `1m` 窗口和 `速率` 口径
- **THEN** 两端展示的池子排序 MUST 基于同一组后端字段
- **AND** 同一池子的手续费、速率和样本状态含义必须一致
