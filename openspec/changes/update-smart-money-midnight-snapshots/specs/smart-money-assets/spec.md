## ADDED Requirements

### Requirement: 聪明钱资产必须保存独立 0 点快照
系统 SHALL 使用独立数据表保存聪明钱钱包每日 0 点资产快照，且该快照不得被日内实时刷新、每日聚合重跑或服务重启后的普通调度覆盖。

#### Scenario: 写入当日 0 点快照
- **WHEN** 系统在某自然日 0 点后采集聪明钱钱包资产
- **THEN** 系统在独立 0 点快照表中为每个活跃聪明钱钱包保存该自然日的资产拆分与总资产
- **AND** 快照按钱包地址、链 ID 和快照日唯一

#### Scenario: 快照已存在时调度重跑
- **GIVEN** 独立 0 点快照表中已存在某钱包某自然日的快照
- **WHEN** 服务重启后普通调度再次尝试采集同一自然日快照
- **THEN** 系统不得覆盖已有快照金额
- **AND** 系统不得把日内最新资产写成该日 0 点快照

### Requirement: 聪明钱 0 点快照必须至少保留 31 天
系统 SHALL 保留最近至少 31 天的聪明钱 0 点资产快照，以支持月度资产趋势、排行榜基准和钱包盈亏日历查询。

#### Scenario: 执行快照保留清理
- **WHEN** 系统执行聪明钱 0 点快照保留清理
- **THEN** 最近 31 天内的 0 点快照仍可查询
- **AND** 超过保留窗口的数据可以被清理

### Requirement: 聪明钱实时排行榜必须使用当日 0 点快照作为基准
系统 MUST 使用当日 0 点资产快照作为聪明钱实时排行榜的基准，并使用最近一次实时资产快照作为当前资产计算差值。

#### Scenario: 请求实时资产排行榜
- **GIVEN** 某钱包存在当日 0 点资产快照和最近实时资产快照
- **WHEN** 客户端请求聪明钱实时资产排行榜
- **THEN** 排行榜条目的 `baseline_total_usd` 等于该钱包当日 0 点资产快照总额
- **AND** `current_total_usd` 等于该钱包最近实时资产快照总额
- **AND** 盈亏或变化值按最近实时资产总额减当日 0 点资产快照总额计算

### Requirement: 聪明钱盈亏日历必须按相邻 0 点快照计算
系统 MUST 使用当日 0 点资产快照与前一自然日 0 点资产快照计算聪明钱盈亏日历中的每日资产变化。

#### Scenario: 查询钱包盈亏日历
- **GIVEN** 某钱包存在连续两天的 0 点资产快照
- **WHEN** 客户端查询该钱包盈亏日历
- **THEN** 第二天的资产变化值等于第二天 0 点总资产减第一天 0 点总资产
- **AND** 系统不得使用第二天日内最新资产替代第二天 0 点快照

#### Scenario: 查询总览盈亏日历
- **GIVEN** 全部活跃聪明钱钱包存在连续两天的 0 点资产快照
- **WHEN** 客户端查询聪明钱总览盈亏日历
- **THEN** 某天的资产变化值等于全部活跃钱包该天 0 点总资产之和减前一日 0 点总资产之和
