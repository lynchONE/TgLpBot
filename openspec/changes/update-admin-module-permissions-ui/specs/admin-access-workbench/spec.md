## MODIFIED Requirements
### Requirement: Web 管理端用户授权管理
系统 SHALL 允许管理员在 `webapp` 与 `miniapp` 管理页查看、搜索、创建或更新用户授权，并能关闭或恢复用户授权。系统 SHALL 支持管理员为用户勾选可访问功能模块、授权所有功能模块或清空模块授权，并在用户授权列表和详情中展示模块授权摘要。

#### Scenario: 管理员查看用户授权列表
- **WHEN** 管理员打开管理页的用户授权入口
- **THEN** 系统返回分页用户授权列表，包含用户身份、授权状态、MiniApp 权限、功能模块授权摘要、有效期、钱包额度、活跃任务额度、备注和更新时间

#### Scenario: 管理员修改用户到期时间
- **WHEN** 管理员提交某用户新的 `active_to`
- **THEN** 系统更新该用户授权到期时间，并保持未提交字段不变

#### Scenario: 管理员勾选用户功能模块
- **WHEN** 管理员在用户授权详情中勾选部分功能模块并保存
- **THEN** 系统只允许该用户访问已勾选模块，并在响应中返回最新模块清单

#### Scenario: 管理员授权所有功能模块
- **WHEN** 管理员点击授权所有功能模块并保存
- **THEN** 系统写入当前全部可授权模块 key，用户后续可访问所有被授权模块

#### Scenario: 管理员清空用户模块授权
- **WHEN** 管理员显式清空用户模块授权并保存
- **THEN** 系统保存空模块清单，用户仍可保留授权记录但不可访问任何普通功能模块

#### Scenario: 管理员关闭用户授权
- **WHEN** 管理员对某用户执行停用授权
- **THEN** 系统写入停用时间和操作管理员，后续用户访问 MiniApp MUST 被拒绝

#### Scenario: 管理员恢复用户授权
- **WHEN** 管理员对已停用用户执行恢复授权
- **THEN** 系统清空停用字段，并按该用户有效期、MiniApp 权限和功能模块授权继续进行访问校验

### Requirement: Web 管理端授权码管理
系统 SHALL 允许管理员在 `webapp` 与 `miniapp` 管理页生成、查看、更新、启用和停用授权码。系统 SHALL 支持管理员为授权码配置默认功能模块，用户兑换授权码后 SHALL 获得该授权码配置的模块权限。

#### Scenario: 管理员生成授权码
- **WHEN** 管理员填写有效期、最大兑换次数、钱包额度、活跃任务额度、MiniApp 权限、功能模块权限和备注并提交
- **THEN** 系统生成唯一授权码，保存授权码配置，并返回明文授权码用于复制分发

#### Scenario: 管理员编辑授权码模块
- **WHEN** 管理员修改授权码的默认功能模块并保存
- **THEN** 系统更新该授权码配置，后续兑换用户 SHALL 使用新的模块清单

#### Scenario: 用户兑换带模块授权的授权码
- **WHEN** 用户兑换有效授权码
- **THEN** 系统创建或更新用户授权，并把授权码的功能模块清单写入用户授权

#### Scenario: 管理员停用授权码
- **WHEN** 管理员停用某个授权码
- **THEN** 系统写入 `disabled_at`，后续用户兑换该授权码 MUST 失败

#### Scenario: 管理员调整授权码额度
- **WHEN** 管理员修改授权码的有效期、兑换次数或额度
- **THEN** 系统只更新管理员提交的字段，并在后续兑换时使用新配置

### Requirement: 管理员 API 鉴权一致性
系统 SHALL 对所有新增管理 API 统一执行 Telegram WebApp initData 验证和管理员身份校验。系统 SHALL 对普通用户功能模块访问执行授权校验，避免仅依赖前端隐藏入口。

#### Scenario: 非管理员访问新增管理 API
- **WHEN** 非管理员用户调用用户授权、授权码或公告管理 API
- **THEN** 系统返回 403，并且不执行任何写入

#### Scenario: initData 无效
- **WHEN** 请求缺少或携带无效 Telegram WebApp initData
- **THEN** 系统返回 401 或 403，并且不执行任何写入

#### Scenario: 用户访问未授权模块
- **WHEN** 已登录用户请求未被授权的功能模块 API
- **THEN** 系统拒绝请求，并返回可识别的无权限错误

## ADDED Requirements
### Requirement: 管理员授权页面易用性
系统 SHALL 在 `webapp` 与 `miniapp` 提供简洁、美观、易操作的管理员授权页面，使管理员能快速搜索对象、查看授权状态、勾选模块权限并确认高影响操作。

#### Scenario: 桌面端编辑用户授权
- **WHEN** 管理员在 `webapp` 打开用户授权工作台并选择一个用户
- **THEN** 页面 SHALL 在同一视图展示用户列表、授权摘要、额度字段、有效期字段和模块权限矩阵，保存操作保持可见

#### Scenario: 移动端编辑用户授权
- **WHEN** 管理员在 `miniapp` 打开用户授权工作台并选择一个用户
- **THEN** 页面 SHALL 使用适合移动端的单列布局展示同等字段，模块勾选控件不得互相遮挡或溢出

#### Scenario: 高影响操作确认
- **WHEN** 管理员停用用户授权、停用授权码、清空全部模块或发布公告
- **THEN** 页面 SHALL 显示明确确认操作，避免误触后立即生效

#### Scenario: 模块批量选择
- **WHEN** 管理员需要快速配置模块权限
- **THEN** 页面 SHALL 提供授权所有功能模块、清空全部模块和按模块分组批量选择的操作
