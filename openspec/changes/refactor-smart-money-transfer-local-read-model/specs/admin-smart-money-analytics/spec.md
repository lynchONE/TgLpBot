## ADDED Requirements

### Requirement: 聪明钱转账日判定必须基于本地转账事件聚合
系统 SHALL 使用本地持久化的聪明钱包转账事件聚合结果判定某自然日是否存在普通转入或转出，不得在读取钱包详情或盈亏日历时再按“今日 0 点到现在”扫描整日区块补转账数据。

#### Scenario: 管理员打开存在当日转账的钱包详情
- **GIVEN** 该钱包当天的普通转账事件已经由监控链路写入本地表
- **WHEN** 管理员打开该钱包详情
- **THEN** 后端从本地转账事件聚合生成转入/转出标识、次数与金额
- **AND** 不再为该请求额外执行当日整段区块扫描

#### Scenario: 盈亏日历展示转账日
- **WHEN** 某自然日在本地转账事件聚合中存在普通转入或转出
- **THEN** 该自然日的聪明钱盈亏值按 0 处理
- **AND** 日历继续展示对应的转入或转出标识
