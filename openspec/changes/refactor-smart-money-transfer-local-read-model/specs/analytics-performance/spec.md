## ADDED Requirements

### Requirement: 聪明钱钱包详情不得在请求时扫描整日区块补转账
后端 MUST 使用本地转账事件读模型为聪明钱钱包详情提供“今天”的转账标识与金额，不得在接口请求期间再把“今日 0 点到现在”映射成整日区块范围进行链上补扫。

#### Scenario: 请求聪明钱钱包详情
- **WHEN** 管理员请求某个聪明钱钱包详情
- **THEN** 后端直接读取本地转账事件聚合、本地日快照和 LP 日统计结果
- **AND** 不得为了补 today 转账信息发起整日区块扫描

#### Scenario: 快链上的今日详情请求
- **GIVEN** 钱包所在链在当天已经产生大量区块
- **WHEN** 管理员请求该钱包详情
- **THEN** 接口仍然返回本地聚合结果
- **AND** 不得因为“maximum block range”这类整日扫链限制而导致转账检测不完整
