## ADDED Requirements

### Requirement: 热门池子支持按头像筛选同代币池子
Web workbench SHALL 允许用户点击热门池子卡片头像，按该池子的非稳定币代币筛选当前链下的相关池子列表。

#### Scenario: 可识别唯一非稳定币
- **WHEN** 用户点击某个热门池子卡片头像，且该池子可识别出唯一非稳定币代币
- **THEN** 热门池子列表切换为该代币的池子筛选结果
- **AND** 当前排序方式与链设置保持不变
- **AND** 界面展示当前代币筛选状态

#### Scenario: 再次点击相同头像
- **WHEN** 用户在已启用该代币筛选的状态下再次点击相同头像
- **THEN** 热门池子列表取消该代币筛选
- **AND** 恢复默认热门池子列表

#### Scenario: 无法识别唯一非稳定币
- **WHEN** 用户点击某个热门池子卡片头像，但系统无法可靠识别该池子的唯一非稳定币代币
- **THEN** 不启用代币筛选
- **AND** 原有池子选择行为不受影响

### Requirement: 热门池子接口支持按代币地址筛选
后端 MUST 允许 `GET /api/hot_pools` 按代币地址返回包含该代币的池子集合。

#### Scenario: 传入有效代币地址筛选
- **WHEN** 调用方在 `GET /api/hot_pools` 中传入有效的代币地址筛选参数
- **THEN** 接口返回当前链、当前 timeframe 下包含该代币地址的池子
- **AND** 返回结构与默认热门池子列表保持兼容

#### Scenario: 未传入代币地址筛选
- **WHEN** 调用方未传入代币地址筛选参数
- **THEN** `GET /api/hot_pools` 继续返回默认热门池子榜单
