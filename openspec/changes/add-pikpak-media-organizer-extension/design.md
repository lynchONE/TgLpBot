# Design: PikPak 媒体整理 Chrome 扩展

## 背景
目标是在不接触用户账号密码的前提下，为已经登录 PikPak Web 的用户提供一个可审阅、可回滚思路清晰的整理工具。工具需要在浏览器内完成扫描、生成计划、执行删除和移动，同时尽量降低因站点结构变更或识别误差导致的误删风险。

PikPak 官方当前公开说明里，WebDAV 仅支持读取，不支持写入、复制和移动，因此本方案不走 WebDAV，而是通过 Chrome 扩展复用用户当前 Web 会话完成操作。

## Goals
- 在 PikPak Web 已登录页面中扫描文件与目录元数据，并识别其中可参与番号整理的视频文件。
- 从文件名中提取番号，并按番号生成重复组。
- 联网查询番号对应的女优元数据，并尽量解析出女优中文名。
- 在番号重复组内优先保留带中文字幕标记的版本。
- 按文件夹清理小于 `100MB` 的小文件。
- 以源目录为单位优先识别目标长视频，并在迁移后清理该目录剩余内容。
- 按添加时间年份进行一级目录归档。
- 按女优中文名称进行二级目录归档。
- 在移动和删除后清理已经腾空的源目录。
- 提供 `Dry Run` 预览与显式确认后的批量执行。
- 对低置信度结果保持跳过，不盲目自动操作。

## Non-Goals
- 不实现 PikPak 登录流程。
- 不解析视频内容本体，不识别封面人物，不做 OCR 或字幕流抽取。
- 不对无法提取番号的文件做自动去重删除。
- 不处理 PikPak 站外客户端或 WebDAV 客户端中的整理操作。

## 架构概览

### 1. Manifest 与脚本分层
- `manifest.json`
  - 声明 PikPak Web 所需的 `host_permissions`
  - 注册 `background.js`、`content-script.js`、`page-hook.js`
  - 提供 `popup.html` 和 `options.html`
- `background.js`
  - 保存用户规则、缓存扫描结果、维护操作计划
  - 执行与节流批量任务
  - 记录最近一次执行结果
- `content-script.js`
  - 与页面通信
  - 读取当前目录上下文
  - 展示必要的页面内提示
- `page-hook.js`
  - 在页面上下文中拦截 `fetch` / `XMLHttpRequest`
  - 抽取 PikPak 文件列表接口返回的数据
  - 在需要执行删除、建目录、移动时复用页面当前会话发起请求
- `popup.js`
  - 承担主要交互入口：扫描、预览、执行、导出结果
- `options.js`
  - 管理关键词规则、女优别名规则、执行策略

### 2. 数据流
1. 用户在 PikPak Web 页面打开扩展弹窗。
2. 扩展请求 `content-script` 识别当前页面是否为可支持的文件列表页。
3. `page-hook` 优先从当前页面已渲染的列表 DOM 中提取文件、目录以及视频项元数据；当 DOM 无法提供足够信息时，再回退到页面内部请求数据。
4. `background` 根据规则生成整理计划。
5. `popup` 展示计划，按“保留 / 删除 / 移动 / 跳过 / 复核”分类。
6. `popup` 同步展示扫描/执行进度、运行日志、待确保目录名称和待删除空目录名称。
7. 用户点击 `Apply` 后，`background` 按顺序执行建目录、移动、删除，并把结果写入本地状态。

## 元数据模型
每个候选文件至少维护以下字段：
- `itemId`
- `parentId`
- `parentPath`
- `name`
- `size`
- `mimeType`
- `createdAt`
- `addedAt`
- `hash`
- `path`
- `isVideo`
- `raw`
- `code`
- `actress`
- `actressNameZh`
- `metadataSource`

其中：
- `name` 和可选 `hash` 共同服务于番号提取与兼容校验。
- `parentId` / `parentPath` 用于“按文件夹删除小文件”策略。
- `addedAt` 用于年份归档；无效时不产生年份移动动作。
- `name` 同时用于番号提取和中文字幕判断。
- `code` 用于在线查询女优信息。
- `actressNameZh` 是最终目录名候选；若缺失，则女优分类降级或跳过。

## 规则引擎

### 1. 中文字幕优先级
对文件名做标准化后，匹配用户配置的字幕关键词集合，并额外识别 `番号-C`、`番号_C` 这类常见中文字幕后缀。默认关键词包含：
- `中字`
- `中文字幕`
- `CHS`
- `C字幕`
- `SUB`

字幕标记仅影响重复组内的保留优先级，不单独触发移动或删除。

### 2. 番号提取与重复判定
- 默认通过正则从文件名中提取标准化番号，优先兼容常见样式：
  - `ABP-123`
  - `IPX123`
  - `SONE-001`
- 标准化规则：
  - 英文字母转大写
  - 自动补全字母与数字间的连接符为 `-`
  - 去除额外空格和多余分隔符
- 自动去重前提：同一扫描范围内，两个或多个文件命中相同标准化番号
- 若重复组内存在字幕标记文件，则优先保留字幕标记文件
- 若多个文件同时满足字幕优先级，则按稳定规则决策保留项：
  - 文件体积更大优先
  - 添加时间更早优先
  - 最后回退到 `itemId` 字典序
- 其余成员进入删除队列
- 若文件无法提取番号，则不参与自动去重

### 3. 按文件夹删除小文件
- 以 `parentId` 为文件夹分组维度
- 默认阈值为 `100MB`
- 处理所有文件类型
- 命中“小文件删除”规则的文件进入删除候选
- 但若该文件已经是某个番号组的最终保留项，则默认不因小文件规则被删除
- 用户可在后续版本中显式放开“保留项也允许按小文件规则删除”，V1 不开放

### 3.1 源目录优先清理
- 当某个源目录内存在至少一个满足迁移条件的目标长视频时，规划器优先将该目录视为“待清理源目录”。
- 待清理源目录中的目标长视频进入移动队列。
- 同一源目录及其子目录中未被选中的其他文件进入删除候选。
- 执行阶段在移动完成后再次确认源目录是否为空；为空则连同空子目录一起删除。

### 4. 在线女优查询
- 背景脚本维护可配置的数据源链路，例如：
  - `providerA`: 番号 -> 女优标准名
  - `providerB`: 女优标准名 -> 中文名
  - `localOverride`: 用户本地中文名覆盖
- 查询顺序：
  1. 使用标准化番号访问在线元数据源
  2. 提取女优标准名、别名或现成中文名
  3. 若未得到中文名，则尝试二次查询或本地覆盖
  4. 若仍失败，则不给出女优目录名
- 扩展必须对在线查询结果做本地缓存，避免重复请求与被限流
- 在线查询失败不应阻断其他整理逻辑，但会影响女优分类能力

### 5. 年份分类
- 从 `addedAt` 解析四位年份
- 识别成功后生成一级目标目录 `YYYY`
- 若年份无效或缺失，则不生成移动动作

### 6. 女优分类
- 若在线查询成功得到女优中文名，则生成二级目录 `YYYY/女优中文名`
- 若只有非中文标准名但存在本地中文覆盖，则使用覆盖后的中文名
- 若没有可靠中文名，则默认退回到年份目录 `YYYY`
- 多女优结果或冲突结果时标记为人工复核，默认不执行女优分类移动

### 7. 计划合成优先级
- 第一步：生成番号重复组并选出每组保留项
- 第二步：对非保留项应用小文件规则
- 第三步：联网查询保留项和未删除项的女优中文名
- 第四步：对保留项和未删除项规划年份/女优目标目录
- 第五步：根据“存在目标长视频的源目录”扩展删除候选，并推导会被腾空的源目录
- 第六步：输出 `保留 / 删除 / 移动 / 跳过 / 复核` 结果

## 执行策略

### 0. 源目录工作模式
- 扫描阶段的目的不是先“全量完成所有分类”，而是识别哪些源目录里存在目标长视频，以及这些目录中哪些剩余文件需要一起清理。
- 预览必须明确展示待清理源目录名称、待创建目标目录名称，以及每个视频最终是按女优目录还是按年份目录归档。

### 1. Dry Run
- 默认只生成计划，不调用删除或移动接口
- 预览列表必须清楚展示：
  - 保留文件
  - 删除文件
  - 删除原因（番号重复、小文件、两者皆命中）
  - 目标目录
  - 待确保目录名称
  - 待删除空目录名称
  - 女优中文名来源
  - 跳过原因
  - 人工复核原因
  - 当前扫描模式、进度与运行日志

### 1.5 Direct Execute
- 扩展提供“直接执行当前目录”入口。
- 该入口不要求用户先停留在预览，但仍必须先生成当前目录对应的整理计划。
- 该入口必须遵循当前“递归扫描”设置：开启时直接递归整理当前目录及其子目录，关闭时只整理当前目录。
- 若快速生成的计划为空，系统必须停止执行并明确提示当前目录没有可操作内容。

### 2. Apply
- 执行前再次弹出确认
- 执行顺序：
  1. 创建缺失的年份目录
  2. 创建缺失的女优目录
  3. 按源目录移动目标长视频到目标目录
  4. 删除源目录中未被选中的剩余文件和重复项
  5. 重新检查候选源目录是否为空，空则删除
- 若 PikPak Web 的删除语义是移入回收站，则扩展遵循站点现有语义，不额外做永久删除
- 任何单条失败都应被记录，不中断整个批次
- 扫描与执行阶段都必须持续输出进度和运行日志，避免用户误以为卡死

## 安全策略
- 扩展不保存账号密码，仅复用当前页面登录状态。
- 删除动作必须满足：
  - 用户点击 `Apply`
  - 文件属于番号重复组中的非保留项，或命中小文件规则且不受保留保护
  - 删除目标不是当前保留项
- 目录删除动作必须满足：
  - 目录属于本次计划识别出的空目录候选
  - 文件移动与删除执行后再次确认目录没有剩余子项
- 对无法提取番号且仅名称相似的文件，只允许标记为“复核”，不得自动删除。
- 若 PikPak 接口字段缺失或结构变化导致核心元数据不可用，扩展必须停止执行并提示页面兼容性问题。

## 兼容与降级
- 优先读取 PikPak 当前页面已渲染的 DOM 列表；若 DOM 无法提供足够字段，再尝试从页面内部请求读取结构化数据。
- 当用户开启递归扫描且页面会话可用时，允许优先使用页面内部请求获取完整递归结果；若页面会话不可用，则必须明确告知当前结果仅覆盖 DOM 已渲染条目。
- 如果页面只返回部分字段，则进入降级模式：
  - 没有可靠文件名：禁用番号识别和女优识别
  - 没有 `parentId`：禁用按文件夹删除小文件
  - 没有 `addedAt`：禁用年份分类
  - 没有可靠 `itemId` / `parentId`：禁用执行模式，仅支持导出计划
  - 在线元数据源失败：禁用女优中文目录，保留年份分类

## 验证方案
- 手工验证：
  - 当前目录扫描
  - 递归扫描
  - 同番号文件组内字幕版本优先保留
  - 同番号且同样命中字幕时保留更大文件
  - 每文件夹小于 `100MB` 文件清理
  - 联网查询番号并解析女优中文名
  - 中文名查询失败时回退到年份目录
  - 年份目录创建与移动
  - 移动与删除后空目录清理
  - 无法分类项跳过
- 回归验证：
  - 非 PikPak 页面不激活
  - 未登录页面不给执行入口
  - 无法提取番号时不触发自动去重删除
  - 小文件规则不删除番号组内已选中的保留项
  - `番号-C` / `番号_C` 可命中中文字幕优先级
  - 目录未空时不得误删目录
  - 在线元数据源失败时不影响去重和年份分类
