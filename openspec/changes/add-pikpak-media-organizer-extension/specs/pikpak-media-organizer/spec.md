## ADDED Requirements

### Requirement: 扩展必须在已登录的 PikPak Web 页面扫描文件与目录元数据
系统 MUST 仅在用户已经登录且当前页面可识别为 PikPak 文件列表页时启用扫描能力，并提取用于整理的文件与目录元数据，同时标记其中可参与番号整理的视频文件。

#### Scenario: 在已登录文件列表页成功扫描
- **GIVEN** 用户已经登录 PikPak Web 且当前页面存在文件列表
- **WHEN** 用户在扩展中触发扫描
- **THEN** 系统必须收集当前扫描范围内的文件与目录元数据
- **AND** 系统必须将每个候选文件标记为可整理、跳过或待复核

#### Scenario: DOM 可用时优先从页面已渲染列表扫描
- **GIVEN** 用户当前打开的 PikPak 页面已经渲染出文件列表
- **WHEN** 用户在扩展中触发扫描
- **THEN** 系统必须优先从当前页面 DOM 中读取可见文件与目录元数据
- **AND** 只有当 DOM 无法提供足够结果时，系统才允许回退到页面内部请求数据

#### Scenario: 递归扫描在缺少页面会话时必须提示结果不完整
- **GIVEN** 用户开启了递归扫描
- **AND** 当前页面尚未捕获到可用于递归遍历的页面会话
- **WHEN** 系统触发扫描
- **THEN** 系统可以回退到当前页面 DOM 已渲染列表生成预览
- **AND** 系统必须明确提示当前结果不包含完整递归范围

#### Scenario: 非 PikPak 页面或未登录页面不启用
- **GIVEN** 用户当前不在 PikPak 文件列表页或页面尚未登录
- **WHEN** 用户打开扩展弹窗
- **THEN** 系统必须禁用扫描与执行入口
- **AND** 系统必须提示当前页面不满足运行条件

### Requirement: 扩展必须按番号识别重复并生成保留与删除计划
系统 MUST 从文件名中提取并标准化番号，将番号一致的文件视为重复组；系统 MUST 在重复组内优先保留带中文字幕标记的版本，并在仍需决策时优先保留更大的文件；系统 MUST 将关键词命中与 `番号-C`、`番号_C` 等常见后缀一起视为中文字幕标记来源。

#### Scenario: 同番号重复组内优先保留中文字幕版本
- **GIVEN** 扫描结果中存在多个提取到相同标准化番号的视频文件
- **AND** 其中至少一个文件名匹配中文字幕关键词
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将带中文字幕标记的文件作为保留项
- **AND** 系统必须将同组其他文件加入删除候选

#### Scenario: 同番号且同样命中字幕时保留更大文件
- **GIVEN** 扫描结果中存在多个提取到相同标准化番号的视频文件
- **AND** 两个或多个文件同时命中中文字幕关键词或同时未命中
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将体积更大的文件作为保留项
- **AND** 系统必须将同组其他文件加入删除候选

#### Scenario: 番号后缀 C 也视为中文字幕标记
- **GIVEN** 某个视频文件名包含标准化番号，且番号后紧跟 `-C`、`_C` 或等价分隔形式
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将该文件视为命中中文字幕标记
- **AND** 系统必须允许该标记参与重复组内的保留优先级决策

#### Scenario: 无法提取番号时禁止自动去重删除
- **GIVEN** 某个视频文件无法从文件名中提取出有效番号
- **WHEN** 系统生成整理计划
- **THEN** 系统不得仅因名称相似或大小相同而自动生成删除动作
- **AND** 系统必须将该文件标记为跳过或待复核

### Requirement: 扩展必须支持按文件夹清理小文件
系统 MUST 支持按文件夹识别并清理小于阈值的文件；V1 默认阈值为 `100MB`，且默认对所有文件类型生效。

#### Scenario: 文件夹内小于 100MB 的文件进入删除候选
- **GIVEN** 某个文件夹内存在体积小于 `100MB` 的文件
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将这些文件标记为小文件删除候选
- **AND** 系统必须在预览中展示命中阈值的原因

#### Scenario: 小文件规则不覆盖重复组保留项
- **GIVEN** 某个文件同时命中“小于 100MB”规则
- **AND** 该文件已经被系统选为某个番号重复组的保留项
- **WHEN** 系统生成整理计划
- **THEN** 系统不得仅因小文件规则删除该文件
- **AND** 系统必须将其继续标记为保留项

### Requirement: 扩展必须联网查询番号对应的女优并优先使用中文名
系统 MUST 支持根据标准化番号访问在线元数据源，查询对应的女优信息；系统 MUST 在能够获得女优中文名时使用中文名作为分类目录名。

#### Scenario: 在线查询成功并得到女优中文名
- **GIVEN** 某个视频文件已经提取出有效番号
- **AND** 在线元数据源成功返回该番号对应的女优中文名
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将该中文名作为女优分类目录名
- **AND** 系统必须在预览中标记该中文名的来源

#### Scenario: 在线查询仅返回非中文名时使用本地中文覆盖或放弃女优分类
- **GIVEN** 某个视频文件已经提取出有效番号
- **AND** 在线元数据源只返回日文名、罗马字名或其他非中文名
- **WHEN** 系统生成整理计划
- **THEN** 系统必须优先尝试本地中文覆盖规则
- **AND** 若仍无法获得可靠中文名，则系统不得生成非中文的女优目录名

### Requirement: 扩展必须按添加时间年份规划目标目录
系统 MUST 根据文件的添加时间解析年份，并将可识别年份的文件规划到对应的一级年份目录中。

#### Scenario: 可以识别添加时间年份时生成年份目录
- **GIVEN** 某个视频文件具备有效的添加时间
- **WHEN** 系统生成整理计划
- **THEN** 系统必须为该文件生成对应的 `YYYY` 一级目录目标
- **AND** 若目标目录不存在，系统必须在执行阶段创建该目录

#### Scenario: 添加时间无效时不做年份移动
- **GIVEN** 某个视频文件缺少添加时间或添加时间无法解析为有效年份
- **WHEN** 系统生成整理计划
- **THEN** 系统不得为该文件生成年份移动动作
- **AND** 系统必须将原因标记为跳过或待复核

#### Scenario: 女优无法可靠识别但年份有效时退回年份目录
- **GIVEN** 某个视频文件具备有效的添加时间
- **AND** 系统无法为该文件得到可靠的女优中文名
- **WHEN** 系统生成整理计划
- **THEN** 系统必须仍然为该文件生成 `YYYY` 年份目录目标
- **AND** 系统不得因为女优识别失败而放弃该文件的年份归档

### Requirement: 扩展必须使用女优中文名规划二级目录
系统 MUST 在获得可靠的女优中文名后，把文件规划到对应的二级目录；若无法可靠获得中文名，系统 MUST 不执行错误归类。

#### Scenario: 获得女优中文名时归入年份下的中文目录
- **GIVEN** 某个视频文件已经识别出有效年份
- **AND** 系统已经获得该文件对应的可靠女优中文名
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将该文件的目标目录规划为 `年份/女优中文名`

#### Scenario: 无法获得可靠中文名时不执行中文女优分类
- **GIVEN** 某个视频文件无法通过在线查询或本地覆盖得到可靠的女优中文名
- **WHEN** 系统生成整理计划
- **THEN** 系统不得为该文件生成错误的中文女优目录移动动作
- **AND** 系统必须将其保留在年份级别目录或标记为待复核

### Requirement: 扩展必须清理因整理而腾空的源目录
系统 MUST 在执行文件移动和删除后识别本次整理腾空的源目录，并在再次确认目录没有剩余子项时删除这些空目录。

#### Scenario: 文件移走后源目录为空则删除
- **GIVEN** 某个源目录中的文件已经在本次整理中全部被移动或删除
- **AND** 执行阶段再次检查该目录时确认没有剩余子项
- **WHEN** 系统执行整理计划
- **THEN** 系统必须将该空目录加入删除执行
- **AND** 系统必须记录该目录删除结果

#### Scenario: 目录仍有残留内容时跳过目录删除
- **GIVEN** 某个源目录被标记为目录清理候选
- **AND** 执行阶段再次检查该目录时仍然发现至少一个剩余子项
- **WHEN** 系统执行整理计划
- **THEN** 系统不得删除该目录
- **AND** 系统必须将该目录记录为跳过而不是失败

#### Scenario: 源目录识别出目标长视频后清理剩余内容
- **GIVEN** 某个源目录内至少存在一个满足迁移条件的目标长视频
- **WHEN** 系统生成整理计划
- **THEN** 系统必须将该视频规划到目标目录
- **AND** 系统必须将同源目录及其子目录中未被选中的剩余文件加入删除候选
- **AND** 系统必须在预览中展示该源目录属于待清理源目录

### Requirement: 扩展必须先预览再执行高风险整理动作
系统 MUST 在执行删除、建目录和移动前提供可审阅的整理计划，并且只有在用户显式确认后才允许执行高风险动作。

#### Scenario: 用户预览并确认后执行整理
- **GIVEN** 系统已经生成包含保留、删除、移动和跳过结果的整理计划
- **WHEN** 用户点击 `Apply` 并确认执行
- **THEN** 系统必须按照计划执行建目录、移动和删除动作
- **AND** 系统必须记录每条动作的执行结果

#### Scenario: 用户直接执行当前目录时遵循递归设置
- **GIVEN** 用户当前位于 PikPak 的某个目录
- **AND** 系统提供“直接执行当前目录”入口
- **WHEN** 用户确认直接执行
- **THEN** 系统必须先生成该目录的整理计划再执行
- **AND** 若递归扫描已开启，系统必须递归整理当前目录及其子目录
- **AND** 若递归扫描已关闭，系统必须只整理当前目录

#### Scenario: 扫描与执行阶段必须持续展示进度和运行日志
- **GIVEN** 系统正在进行扫描或执行整理
- **WHEN** 用户查看扩展弹窗
- **THEN** 系统必须展示当前阶段的进度信息
- **AND** 系统必须持续输出运行日志，说明当前正在处理的步骤或对象

#### Scenario: 接口兼容性不足时停止执行
- **GIVEN** 扩展在当前 PikPak 页面无法获得执行所需的关键字段或接口行为异常
- **WHEN** 用户尝试执行整理计划
- **THEN** 系统必须停止高风险执行
- **AND** 系统必须向用户提示当前页面版本不兼容或元数据不足
