## ADDED Requirements

### Requirement: 按代币从索引数据源发现大额加池钱包
系统 SHALL 提供基于代币地址的大额加池钱包候选预览能力。用户 MUST 能输入链、代币地址、最低加池 USD 金额、时间窗口和返回数量，系统 MUST 通过外部索引数据源返回符合条件的钱包候选及对应交易证据。

#### Scenario: 用户按代币和金额筛选候选钱包
- **WHEN** 用户输入 BSC 代币地址、最低加池金额 500U、时间窗口和数量限制后请求预览
- **THEN** 系统 MUST 查询已配置索引数据源中的该代币相关流动性事件
- **AND** 仅返回加池 USD 金额大于等于 500 的候选钱包
- **AND** 每个候选 MUST 包含钱包地址、加池金额、交易时间、交易哈希、实际池子地址、金额来源和数据源
- **AND** 系统 MUST 标记该钱包是否已在聪明钱监控列表中

#### Scenario: 主索引数据源未配置
- **WHEN** 用户请求筛选预览但系统未配置可用的流动性事件索引数据源
- **THEN** 系统 MUST 返回明确配置错误
- **AND** MUST NOT 执行 RPC 查询
- **AND** MUST NOT 返回伪成功空结果

#### Scenario: 辅助 DEX 终端没有加池钱包接口
- **WHEN** GMGN、DexScreener、GeckoTerminal 或 DEXTools 只能提供代币、池子、交易或价格信息
- **THEN** 系统 MAY 使用这些数据补充展示或池子发现
- **AND** MUST NOT 将这些辅助数据伪装成 LP 加池钱包证据

#### Scenario: 加池金额字段缺失
- **WHEN** 某条加池事件无法得到明确可信的 USD 金额
- **THEN** 系统 MUST 排除该条事件并在响应中记录排除原因或返回解析错误
- **AND** MUST NOT 使用 0、token 数量或默认金额替代 USD 金额

### Requirement: 候选钱包批量导入聪明钱
系统 SHALL 允许用户将预览后确认的钱包批量导入聪明钱监控列表。导入 MUST 写入现有 `monitored_wallets` 监控体系，来源标记为代币加池索引源筛选，并保留来源代币地址。

#### Scenario: 用户确认导入候选钱包
- **WHEN** 用户从预览结果中选择一个或多个钱包并提交导入
- **THEN** 系统 MUST 校验每个钱包地址和来源代币地址
- **AND** 新钱包 MUST 创建为启用状态的聪明钱监控钱包
- **AND** 钱包来源 MUST 标记为 `token_liquidity_indexer`
- **AND** 来源代币地址 MUST 写入该钱包的来源上下文

#### Scenario: 导入已存在的钱包
- **WHEN** 用户导入的钱包已经存在于聪明钱监控列表
- **THEN** 系统 MUST 不创建重复记录
- **AND** 如果该钱包已停用，系统 MUST 将其重新启用
- **AND** 响应 MUST 返回已存在、重新启用和新创建的钱包数量

#### Scenario: 导入参数包含非法钱包
- **WHEN** 用户提交的钱包列表中包含非法地址
- **THEN** 系统 MUST 拒绝导入请求或在响应中明确列出非法地址
- **AND** MUST NOT 静默丢弃非法地址后伪装为全部成功

### Requirement: MiniApp 提供移动端筛选导入入口
MiniApp 的聪明钱钱包视图 SHALL 提供移动端筛选导入入口。入口 MUST 支持输入代币地址和筛选条件、预览候选、勾选钱包、执行批量导入，并展示导入结果。

#### Scenario: MiniApp 用户执行筛选导入
- **WHEN** 用户在 MiniApp 打开聪明钱钱包视图并点击筛选导入
- **THEN** 前端 MUST 展示适配移动端的输入和候选预览界面
- **AND** 用户点击预览后 MUST 展示候选钱包列表和交易证据
- **AND** 用户确认导入后 MUST 刷新聪明钱钱包列表

### Requirement: WebApp 提供桌面端筛选导入入口
WebApp 的聪明钱钱包视图 SHALL 提供桌面端筛选导入入口。入口 MUST 支持输入代币地址和筛选条件、表格化预览候选、批量勾选、执行导入，并展示导入结果。

#### Scenario: WebApp 用户执行筛选导入
- **WHEN** 用户在 WebApp 打开聪明钱钱包视图并点击筛选导入
- **THEN** 前端 MUST 展示适配桌面端的表单、候选表格和批量操作
- **AND** 用户点击预览后 MUST 展示候选钱包列表、金额来源、数据源、交易哈希和实际池子
- **AND** 用户确认导入后 MUST 刷新聪明钱钱包列表

### Requirement: 权限与错误处理
系统 SHALL 对筛选预览和批量导入执行聪明钱模块权限校验，并对外部数据源、金额计算错误返回明确错误或排除原因。

#### Scenario: 用户没有聪明钱权限
- **WHEN** 用户没有聪明钱模块权限但请求筛选预览或导入
- **THEN** 系统 MUST 拒绝请求
- **AND** 前端 MUST 展示权限不足提示
