## ADDED Requirements

### Requirement: MiniApp 特别关注动态模块必须展示关注钱包的 LP 操作记录
MiniApp 聪明钱模块 MUST 提供一个“特别关注”视图，让用户查看自己已特别关注的钱包，并能进入某个钱包最近的 LP 操作记录。操作记录 MUST 至少包含加 LP 和撤 LP 两类事件，并展示钱包、池子或交易对、操作方向、金额、区间、发生时间和交易哈希。

#### Scenario: 查看特别关注钱包列表
- **GIVEN** 用户已经添加一个或多个特别关注钱包
- **WHEN** 用户进入聪明钱的“特别关注”视图
- **THEN** 系统 MUST 展示这些特别关注钱包
- **AND** 每个钱包 MUST 可点击进入最近操作记录

#### Scenario: 查看某个特别关注钱包的操作记录
- **GIVEN** 用户点击一个特别关注钱包
- **WHEN** 系统加载该钱包最近 LP 事件
- **THEN** 系统 MUST 只展示该钱包的 `add` 和 `remove` LP 事件
- **AND** 记录 MUST 按发生时间倒序排列

#### Scenario: 只能读取当前用户关注范围
- **GIVEN** 钱包 A 只被用户甲特别关注，钱包 B 只被用户乙特别关注
- **WHEN** 用户甲打开特别关注动态
- **THEN** 系统 MUST 只返回钱包 A 的操作记录
- **AND** 系统 MUST NOT 返回钱包 B 的操作记录

#### Scenario: 无特别关注钱包时展示空状态
- **GIVEN** 用户尚未添加任何特别关注钱包
- **WHEN** 用户进入“特别关注”视图
- **THEN** 系统 MUST 展示空状态
- **AND** 系统 MUST 引导用户到钱包视图添加特别关注
