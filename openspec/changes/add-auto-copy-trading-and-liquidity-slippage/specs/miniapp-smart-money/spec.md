## ADDED Requirements

### Requirement: 聪明钱模块提供自动跟单页签
MiniApp SHALL 在聪明钱模块中提供独立的“自动跟单”页签，用于管理用户的聪明钱 LP 自动跟单配置。

#### Scenario: 用户进入自动跟单页签
- **WHEN** 用户打开聪明钱模块并切换到“自动跟单”
- **THEN** 系统 MUST 展示当前用户已配置的跟单钱包、启用状态、金额模式、延时模式和撤仓跟单状态

#### Scenario: 用户没有自动跟单配置
- **WHEN** 用户首次进入“自动跟单”页签且没有任何配置
- **THEN** 系统 MUST 展示可创建新配置的空状态
- **AND** MUST NOT 自动启用任何跟单行为

### Requirement: 自动跟单配置支持目标钱包与金额模式
系统 SHALL 允许用户为每个自动跟单配置设置目标钱包和开仓跟单金额模式。金额模式 MUST 支持固定金额和按被跟单开仓金额比例计算。

#### Scenario: 固定金额跟单
- **WHEN** 用户选择固定金额模式并填写单次跟单 USDT 金额
- **THEN** 系统 MUST 在目标钱包发生新开仓事件时按该固定金额创建跟单开仓作业

#### Scenario: 比例金额跟单
- **WHEN** 用户选择比例金额模式并填写比例
- **AND** 目标钱包发生新开仓事件且事件包含可用开仓 USD 金额
- **THEN** 系统 MUST 按 `目标开仓金额 * 比例` 计算本次跟单金额

#### Scenario: 比例金额缺少目标开仓金额
- **WHEN** 用户选择比例金额模式
- **AND** 目标钱包新开仓事件缺少可用开仓 USD 金额
- **THEN** 系统 MUST 将本次跟单作业标记为失败并记录原因
- **AND** MUST NOT 使用默认金额、0 金额或固定金额兜底

### Requirement: 自动跟单支持立即执行与延时执行
系统 SHALL 允许用户配置自动跟单开仓的执行时机，支持立即跟单和固定延时跟单。

#### Scenario: 立即跟单
- **WHEN** 用户选择立即跟单
- **AND** 目标钱包发生新开仓事件
- **THEN** 系统 MUST 尽快创建并执行跟单开仓作业

#### Scenario: 延时跟单
- **WHEN** 用户选择延时跟单并填写延时秒数
- **AND** 目标钱包发生新开仓事件
- **THEN** 系统 MUST 将跟单开仓作业计划到事件发现时间加延时秒数之后执行

#### Scenario: 延时参数非法
- **WHEN** 用户保存的延时秒数不满足系统允许范围
- **THEN** 系统 MUST 拒绝保存该自动跟单配置

### Requirement: 自动跟单支持新建仓位区间上移
系统 SHALL 允许用户在自动跟单配置中设置 `range_shift_grids`，用于将新建跟单仓位的 tick 区间整体向更高价格方向移动指定数量的 `tickSpacing` 格子。

#### Scenario: 宽区间按配置上移
- **WHEN** 目标钱包发生新建仓位事件
- **AND** 自动跟单配置的 `range_shift_grids` 大于 0
- **AND** 目标仓位区间宽度大于 2 个 `tickSpacing` 格子
- **THEN** 系统 MUST 在创建跟单仓位前，将 `tick_lower` 和 `tick_upper` 同时增加 `range_shift_grids * tickSpacing`
- **AND** MUST 保持目标仓位映射仍指向原始被跟单仓位，确保后续加仓和撤仓能关联同一跟单仓位

#### Scenario: 窄区间不上移
- **WHEN** 目标钱包发生新建仓位事件
- **AND** 目标仓位区间宽度小于或等于 2 个 `tickSpacing` 格子
- **THEN** 系统 MUST 使用目标事件原始 tick 区间创建跟单仓位
- **AND** MUST NOT 因区间上移配置把跟单仓位移出原目标区间

#### Scenario: 非法区间上移配置
- **WHEN** 用户保存的 `range_shift_grids` 小于 0 或超过系统允许上限
- **THEN** 系统 MUST 拒绝保存该自动跟单配置

### Requirement: 自动跟单支持执行钱包池
系统 SHALL 允许用户为自动跟单配置设置多个执行钱包，并选择固定、轮询或随机模式。系统 MUST 在新建跟单仓位时按配置选择执行钱包，并 MUST 在后续加仓和撤仓时沿用该跟单仓位创建时的钱包。

#### Scenario: 轮询选择新建仓位钱包
- **WHEN** 自动跟单配置包含多个执行钱包且选择轮询模式
- **AND** 目标钱包连续产生多个新建仓位事件
- **THEN** 系统 MUST 按执行钱包池顺序为每个新建跟单仓位选择钱包
- **AND** MUST 将本次选择的钱包记录到跟单作业和跟单任务映射

#### Scenario: 加仓沿用原钱包
- **WHEN** 目标钱包对已有仓位加仓
- **AND** 系统已存在该目标仓位对应的跟单仓位映射
- **THEN** 系统 MUST 使用原跟单仓位创建时的钱包执行加仓
- **AND** MUST NOT 因随机或轮询模式切换到其他钱包

#### Scenario: 非法执行钱包池
- **WHEN** 用户保存的执行钱包池为空、包含不属于当前用户的钱包，或超过系统允许上限
- **THEN** 系统 MUST 拒绝保存该自动跟单配置

### Requirement: 自动跟单支持通知配置
系统 SHALL 允许用户为每个自动跟单配置设置是否发送通知和通知强度，并复用系统 Bark 通知能力。通知发送失败 MUST NOT 改变跟单作业的执行结果。

#### Scenario: 作业终态通知
- **WHEN** 自动跟单作业进入成功、失败或跳过终态
- **AND** 该配置开启 Bark 通知且用户 Bark 已配置可用
- **THEN** 系统 MUST 按配置强度发送包含作业动作、状态、金额、执行钱包和失败原因的 Bark 通知

#### Scenario: 通知不可用
- **WHEN** 自动跟单配置开启通知但用户 Bark 未启用、未配置或发送失败
- **THEN** 系统 MUST 记录日志
- **AND** MUST NOT 阻塞作业创建、作业执行或事件流展示

### Requirement: 自动跟单支持止盈止损停止
系统 SHALL 允许用户为自动跟单配置设置止盈金额和止损金额。系统 MUST 按该配置下跟单任务的已实现 PnL 与未实现 PnL 汇总判断是否停止继续跟单；触发后 MUST 停用该配置，但 MUST NOT 自动平掉已有仓位。

#### Scenario: 止盈触发停止跟单
- **WHEN** 配置的止盈金额大于 0
- **AND** 该配置下跟单总盈亏大于或等于止盈金额
- **THEN** 系统 MUST 将该自动跟单配置停用
- **AND** MUST 记录触发时间、触发原因和触发时总盈亏

#### Scenario: 止损触发停止跟单
- **WHEN** 配置的止损金额大于 0
- **AND** 该配置下跟单总盈亏小于或等于负的止损金额
- **THEN** 系统 MUST 将该自动跟单配置停用
- **AND** MUST 记录触发时间、触发原因和触发时总盈亏

#### Scenario: PnL 不完整时不触发
- **WHEN** 系统无法计算某个未平仓跟单任务的当前 PnL
- **THEN** 系统 MUST 在状态视图中展示 PnL 错误
- **AND** MUST NOT 使用不完整 PnL 触发止盈或止损停止

### Requirement: 自动跟单提供状态和收益视图
系统 SHALL 在自动跟单页面展示每个配置当前跟单状态，包括运行状态、执行钱包池、开仓数量、已平仓数量、已实现 PnL、未实现 PnL、总 PnL 和风控触发信息。我的资产趋势 MUST 支持切换到跟单盈利曲线。

#### Scenario: 查看跟单状态
- **WHEN** 用户进入自动跟单状态视图
- **THEN** 系统 MUST 展示每个自动跟单配置的当前盈亏、仓位数量和风控阈值
- **AND** MUST 在配置被止盈或止损停止时展示停止原因和触发盈亏

#### Scenario: 查看跟单盈利曲线
- **WHEN** 用户进入“我的资产”资产趋势区域并选择跟单盈利
- **THEN** 系统 MUST 展示该用户自动跟单任务已平仓收益的累计曲线

### Requirement: 自动跟单支持撤仓跟单配置
系统 SHALL 允许用户配置是否跟随目标钱包撤仓。撤仓跟单开启时，系统 MUST 仅对该自动跟单配置创建的对应跟单仓位执行撤仓。

#### Scenario: 撤仓跟单关闭
- **WHEN** 用户关闭撤仓跟单
- **AND** 目标钱包发生撤仓事件
- **THEN** 系统 MUST 不发起用户仓位撤仓
- **AND** MAY 记录该事件已按配置跳过

#### Scenario: 撤仓跟单开启且存在对应跟单仓位
- **WHEN** 用户开启撤仓跟单
- **AND** 目标钱包发生撤仓事件
- **AND** 系统找到该配置此前创建的对应跟单仓位
- **THEN** 系统 MUST 为该跟单仓位创建撤仓作业

#### Scenario: 撤仓跟单开启但没有对应跟单仓位
- **WHEN** 用户开启撤仓跟单
- **AND** 目标钱包发生撤仓事件
- **AND** 系统找不到该配置创建的对应跟单仓位
- **THEN** 系统 MUST 将撤仓跟单作业标记为失败或跳过并记录原因
- **AND** MUST NOT 撤出用户手动仓位、AutoLP 仓位或其他跟单配置创建的仓位

### Requirement: 自动跟单只处理启用后的新事件
系统 MUST 在自动跟单配置启用后只处理后续新发生的聪明钱 LP 事件，不得默认回放历史事件。

#### Scenario: 用户启用自动跟单配置
- **WHEN** 用户启用某个目标钱包的自动跟单配置
- **THEN** 系统 MUST 记录当前事件游标或区块位置
- **AND** 后续 worker 只处理该游标之后的新事件

#### Scenario: 用户停用自动跟单配置
- **WHEN** 用户停用某个自动跟单配置
- **THEN** 系统 MUST 停止为该配置生成新的开仓或撤仓跟单作业
- **AND** 已创建但尚未执行的作业 MUST 按停用状态跳过或取消
