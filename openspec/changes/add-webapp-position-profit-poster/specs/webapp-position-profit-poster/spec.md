## ADDED Requirements

### Requirement: WebApp 仓位卡片支持生成收益海报
系统 MUST 在 `webapp` 的运行中仓位卡片中提供“收益图”入口，允许用户为单个任务生成收益海报。

#### Scenario: 用户从仓位卡片进入收益海报预览
- **GIVEN** 用户已经登录 `webapp` 且可见一个运行中的仓位任务
- **WHEN** 用户点击该任务上的“收益图”入口
- **THEN** 系统会拉取该任务的收益海报数据
- **AND** 打开一个海报预览弹窗
- **AND** 用户可以下载生成后的 PNG 图片

### Requirement: 收益海报必须包含完整身份与收益摘要
系统 MUST 在收益海报中展示当前用户身份、仓位身份以及当前收益摘要信息，确保图片可独立阅读。

#### Scenario: 海报展示核心摘要字段
- **GIVEN** 某个运行中的任务存在有效的仓位数据
- **WHEN** 系统生成该任务的收益海报
- **THEN** 海报中必须包含用户头像和名字
- **AND** 海报中必须包含交易对名称
- **AND** 海报中必须包含本次投入金额
- **AND** 海报中必须包含未实现利润
- **AND** 海报中必须包含盈利百分比
- **AND** 海报中必须包含主题代币名称与代币头像（若可获取）

### Requirement: 收益海报必须展示开单以来走势
系统 MUST 为收益海报提供一条从本次开单时间到当前时间的走势曲线，用于表达开单以来的表现变化。

#### Scenario: 成功获取 OKX 行情数据时绘制走势曲线
- **GIVEN** 系统能够获取该任务主题代币自开单以来的 OKX K 线数据
- **WHEN** 系统生成收益海报
- **THEN** 海报中必须绘制一条开单以来走势曲线
- **AND** 该曲线以开单时刻为基准进行归一化展示

#### Scenario: 行情数据缺失时降级生成海报
- **GIVEN** 系统未能获取该任务主题代币的走势数据
- **WHEN** 用户生成收益海报
- **THEN** 系统仍必须生成可下载海报
- **AND** 海报中的走势区域必须展示清晰的缺省提示

### Requirement: 系统必须对头像与代币信息缺失进行降级
系统 MUST 在用户头像或代币头像缺失时继续生成海报，而不是直接失败。

#### Scenario: 用户头像缺失时使用首字母头像
- **GIVEN** 当前用户没有可用头像 URL
- **WHEN** 系统生成收益海报
- **THEN** 海报中必须使用用户名字或用户名首字母生成占位头像

#### Scenario: OKX 代币基础信息缺失时使用符号占位
- **GIVEN** OKX 未返回主题代币头像或名称
- **WHEN** 系统生成收益海报
- **THEN** 海报中必须回退到仓位已有的代币符号或交易对文本
- **AND** 代币头像区域必须使用符号首字母占位
