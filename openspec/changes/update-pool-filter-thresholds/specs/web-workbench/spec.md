## ADDED Requirements

### Requirement: 热门池子筛选必须支持排除高费率池子
系统 SHALL 在 Web Workbench 的热门池子筛选中提供最大费率阈值，用户启用后 MUST 排除费率高于该阈值的池子。

#### Scenario: 用户排除高费率热门池子
- **WHEN** 用户在热门池子筛选中设置“排除费率高于 1%”
- **THEN** 热门池子列表不得展示 `fee_rate > 1` 的池子
- **AND** 其他已启用的热门池子筛选条件继续同时生效

#### Scenario: 用户清空最大费率阈值
- **WHEN** 用户清空“排除费率高于”输入并应用筛选
- **THEN** 热门池子列表恢复为不按最大费率过滤
- **AND** 已设置的其他筛选条件继续保留

### Requirement: 聪明钱池子视图必须支持池子级筛选
系统 SHALL 在 Web Workbench 的聪明钱池子视图提供筛选入口，支持按池子当前聪明钱仓位金额和池子费率过滤池子列表。

#### Scenario: 用户按聪明钱仓位金额筛选池子
- **WHEN** 用户设置“聪明钱仓位金额大于等于 1000U”
- **THEN** 聪明钱池子视图只展示当前聪明钱仓位金额不低于 1000U 的池子

#### Scenario: 用户按池子费率筛选聪明钱池子
- **WHEN** 用户设置“池子费率小于等于 1%”
- **THEN** 聪明钱池子视图不得展示费率高于 1% 的池子
- **AND** 缺少费率数据的池子不得通过该费率筛选
