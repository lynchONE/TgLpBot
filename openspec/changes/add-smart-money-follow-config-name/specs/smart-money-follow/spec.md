## ADDED Requirements

### Requirement: 自动跟单配置支持任务名称
系统 SHALL 允许用户为每条自动跟单配置保存一个可选任务名称，用于说明该配置监控的聪明钱或策略意图。任务名称 MUST 仅作为展示和识别信息，不得影响跟单触发、金额计算、执行钱包选择或撤仓匹配逻辑。

#### Scenario: 保存带任务名称的配置
- **WHEN** 用户保存自动跟单配置并填写合法任务名称
- **THEN** 系统 MUST 将规范化后的任务名称保存到该配置
- **AND** 后续查询配置列表 MUST 返回该任务名称

#### Scenario: 清空任务名称
- **GIVEN** 自动跟单配置已有任务名称
- **WHEN** 用户编辑配置并清空任务名称
- **THEN** 系统 MUST 清空该配置的任务名称
- **AND** MUST NOT 使用目标钱包地址生成并保存伪默认名称

#### Scenario: 任务名称超长
- **WHEN** 用户保存超过允许长度的任务名称
- **THEN** 系统 MUST 拒绝保存并返回明确错误
- **AND** MUST NOT 截断后静默保存

#### Scenario: 跟单执行不受任务名称影响
- **GIVEN** 两条自动跟单配置除了任务名称不同以外其余触发和执行参数一致
- **WHEN** 目标钱包产生新的 LP 开仓事件
- **THEN** 系统 MUST 按现有触发规则创建跟单任务
- **AND** MUST NOT 将任务名称参与去重、金额计算或仓位匹配
