## ADDED Requirements
### Requirement: 聪明钱池子视图 MUST 保持交互能力并改善慢查询体验
MiniApp 与 WebApp 的聪明钱池子视图 MUST 保持现有活跃池子、收益火焰图、仓位详情和快捷跟单入口能力，同时后端优化不得改变开仓预览、风控和提交开仓的核心流程。

#### Scenario: 用户打开活跃池子
- **WHEN** 用户进入聪明钱池子视图的活跃池子子页
- **THEN** 页面 MUST 继续展示池子列表、筛选、分页、详情入口和快捷跟单入口
- **AND** 后端不得因链上状态修复阻塞列表响应

#### Scenario: 用户打开收益火焰图
- **WHEN** 用户进入收益火焰图子页
- **THEN** 页面 MUST 继续展示手续费、速率、窗口、分页和样本状态
- **AND** 过期手续费快照刷新不得阻塞当前列表响应

#### Scenario: 用户从聪明钱池子快捷跟单
- **WHEN** 用户点击活跃池子或收益火焰图中的快捷跟单
- **THEN** 系统 MUST 继续进入现有开仓流程
- **AND** 本次性能优化 MUST NOT 修改开仓预览、授权、交易提交和撤仓逻辑
