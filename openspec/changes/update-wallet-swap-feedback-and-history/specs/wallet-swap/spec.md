## ADDED Requirements
### Requirement: 钱包一键兑换 MUST 提供明确的完成反馈
当用户在 `webapp` 发起钱包一键兑换并且后端已经确认交易成功后，系统 MUST 向用户展示明确的完成结果，而不是仅展示“已提交”或静默结束。

#### Scenario: 兑换成功后返回完成态结果
- **WHEN** 用户在 `webapp` 提交一笔钱包单币兑换且后端确认交易成功
- **THEN** 执行接口 MUST 返回成功结果
- **AND** MUST 返回交易哈希
- **AND** MUST 以“兑换已完成”语义返回消息
- **AND** SHOULD 返回完成时间、浏览器链接和实际到账数量

#### Scenario: 兑换成功后清理前端执行态
- **GIVEN** 用户刚刚完成一笔钱包一键兑换
- **WHEN** 页面收到成功响应
- **THEN** 页面 MUST 关闭确认弹层
- **AND** MUST 清理执行中状态和旧报价状态
- **AND** MUST 允许用户立即开始下一笔兑换输入

### Requirement: 钱包一键兑换 MUST 提供当前钱包最近兑换历史
系统 MUST 为当前选中钱包提供最近兑换记录查询与展示能力，帮助用户在同一面板内回看刚完成的兑换。

#### Scenario: 当前钱包存在最近兑换记录
- **GIVEN** 当前钱包已有至少一笔确认成功的钱包一键兑换记录
- **WHEN** 页面请求最近兑换历史
- **THEN** 后端 MUST 按时间倒序返回当前钱包的兑换记录
- **AND** 每条记录 MUST 包含兑换方向、输入输出数量、交易状态、创建时间和交易哈希
- **AND** 页面 MUST 提供可跳转的 Tx 链接

#### Scenario: 当前钱包暂无兑换记录
- **GIVEN** 当前钱包还没有钱包一键兑换记录
- **WHEN** 页面请求最近兑换历史
- **THEN** 后端 MUST 返回空列表
- **AND** 页面 MUST 展示明确的空状态提示
