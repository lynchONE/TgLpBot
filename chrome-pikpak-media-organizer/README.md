# PikPak 媒体整理器

这是一个独立的 Chrome 扩展项目，用于在你已经登录的 `PikPak Web` 页面里做这几件事：

- 按番号去重
- 重复组优先保留带中文字幕标记的版本，并把 `番号-C` / `番号_C` 视为中文字幕信号之一
- 每个原始文件夹内删除小于 `100MB` 的文件
- 按“添加时间”的年份分类
- 按女优中文名分类
- 视频移走或删掉后，自动清理已经腾空的原始文件夹
- 直接在 PikPak Web 会话里创建目录、移动文件、删除文件
- 扫描阶段优先读取当前页面已渲染的 DOM 列表；只有 DOM 拿不到结果时才回退到页面内部请求

## 当前实现边界

- 扩展不会帮你登录 PikPak，也不会保存账号密码。
- 扩展依赖你当前浏览器里已经登录好的 PikPak Web 会话。
- 去重标准是“标准化后的番号一致”。
- 删除走 `files:batchDelete`，不是回收站模式。
- 女优中文名优先通过联网查询获得；默认使用的在线来源有：
  - `MissAV(Jina 代理)`
  - `DuckDuckGo HTML 搜索`
  - `Jable(Jina 代理)` 作为原名补充
  - `JAV Database(Jina 代理)` 作为原名补充
- 如果拿不到女优中文名，默认跳过该文件，不移动；可在设置页改成“仅按年份分类”。

## 加载方式

1. 打开 Chrome 的扩展管理页 `chrome://extensions`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择当前目录：`chrome-pikpak-media-organizer/`

## 使用步骤

1. 打开并登录 `https://mypikpak.com/drive`
2. 安装扩展后，先刷新一次 PikPak 标签页
3. 打开扩展弹窗，点击“扫描并生成预览”
4. 检查“删除 / 移动 / 跳过 / 已就位”预览
5. 确认无误后点击“执行计划”

## 设置项

- `递归扫描`：是否扫描当前目录下的全部子目录
- `小文件阈值`：默认 `100MB`
- `字幕关键词`：用于重复组优先保留中文字幕版本
- `元数据源顺序`：控制女优解析的 provider 优先级
- `番号到女优中文名覆盖`：当在线结果不稳定时，可以手动补充

## 兼容性说明

- 扫描时会优先读取当前页面已经渲染出来的文件列表，所以即使页面内部请求上下文暂时不完整，通常也能先做预览。
- 执行移动、删除、建目录时仍需要复用 PikPak Web 当前页面真实发出的 drive 请求来识别会话上下文。
- 如果扩展提示需要先刷新页面捕获会话，通常只需要刷新一次 PikPak 页面。
- PikPak Web 的私有接口和站点结构后续可能变化，变化后可能需要同步调整 `page-hook.js`。
